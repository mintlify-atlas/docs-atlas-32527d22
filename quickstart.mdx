# Quick start

This guide will get you from zero to running your first app behind OpenEnvX's local proxy in under 5 minutes.

## What you'll build

You'll start a simple web server and access it at `https://myapp.localhost:1355` using OpenEnvX's automatic proxy and TLS.

<Steps>
  <Step title="Create a simple server">
    First, let's create a basic Node.js server to proxy. Create a file called `server.js`:
    
    ```javascript server.js
    const http = require('http');
    
    const server = http.createServer((req, res) => {
      res.writeHead(200, { 'Content-Type': 'text/plain' });
      res.end('Hello from OpenEnvX!\n');
    });
    
    const PORT = process.env.PORT || 3000;
    server.listen(PORT, () => {
      console.log(`Server running on port ${PORT}`);
    });
    ```
    
    <Info>
    You can use any web server or framework. OpenEnvX works with Node.js, Python, Go, Ruby, or any HTTP server.
    </Info>
  </Step>
  
  <Step title="Run with OpenEnvX proxy">
    Start your server with OpenEnvX using the `proxy run` command:
    
    ```bash
    openenvx proxy run myapp -- node server.js
    ```
    
    This command does several things:
    - Starts the proxy daemon on port 1355 (if not already running)
    - Generates TLS certificates for `*.localhost`
    - Runs your command (`node server.js`)
    - Registers a route from `myapp.localhost` to your server's port
    
    <Note>
    The `--` separator is required to separate the app name from the command. Everything after `--` is your app's start command.
    </Note>
  </Step>
  
  <Step title="Access your app">
    Open your browser and navigate to:
    
    ```
    https://myapp.localhost:1355
    ```
    
    You should see "Hello from OpenEnvX!" in your browser.
    
    <Warning>
    Your browser may show a certificate warning since OpenEnvX uses self-signed certificates. This is expected for local development. Click "Advanced" and "Proceed" to continue.
    </Warning>
  </Step>
  
  <Step title="Run multiple services">
    OpenEnvX really shines when running multiple services. Let's add another one.
    
    In a new terminal, start a second service:
    
    ```bash
    openenvx proxy run api -- python -m http.server 8000
    ```
    
    Now you have two services running:
    - `https://myapp.localhost:1355` → your Node.js server
    - `https://api.localhost:1355` → your Python server
    
    Both on the same port (1355), different subdomains!
  </Step>
  
  <Step title="Stop services">
    When you're done, stop your services:
    
    - Press `Ctrl+C` in each terminal running a service to stop it
    - The route is automatically removed when the process exits
    
    To stop the proxy daemon entirely:
    
    ```bash
    openenvx proxy stop
    ```
  </Step>
</Steps>

## Understanding the command

Let's break down what happens when you run:

```bash
 openenvx proxy run myapp -- node server.js
```

1. **`openenvx proxy run`**: Starts the proxy run command
2. **`myapp`**: The name/subdomain for your app (becomes `myapp.localhost`)
3. **`--`**: Required separator between name and command
4. **`node server.js`**: Your application's start command

## Common patterns

<CodeGroup>

```bash Node.js
# Run a Next.js dev server
openenvx proxy run frontend -- npm run dev

# Run a custom Node script
openenvx proxy run backend -- node src/index.js
```

```bash Python
# Run a Flask app
openenvx proxy run api -- python app.py

# Run a Django dev server
openenvx proxy run web -- python manage.py runserver
```

```bash Go
# Run a Go server
openenvx proxy run service -- go run main.go
```

```bash Ruby
# Run a Rails server
openenvx proxy run app -- rails server
```

</CodeGroup>

## Handling port conflicts

If you try to register a route that already exists, you'll get an error:

```bash
openenvx proxy run myapp -- node server.js
# Error: route "myapp" already registered (use --force to override)
```

To override an existing route, use the `--force` flag:

```bash
openenvx proxy run myapp --force -- node server.js
```
