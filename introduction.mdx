# Introduction to OpenEnvX

OpenEnvX is a local-first runtime for development that provides a powerful proxy server, service management, and environment integration. Built with agentic workflows in mind, it gives you complete control over your local development environment.

## The problem it solves

Modern development often requires running multiple services locally, each needing their own domain and port. Managing these manually is tedious and error-prone:

- Remembering which port each service runs on
- Manually updating environment variables
- No easy way to share development URLs
- Complex local DNS or hosts file configuration

OpenEnvX solves this by providing **automatic `*.localhost` domain routing** with zero DNS configuration. Every service gets its own subdomain at `https://myapp.localhost:1355`, all on the same port, with automatic TLS certificates.

## Built for agents and humans

OpenEnvX is designed as a control plane that works equally well for you or an AI agent:

- Start/stop services from the CLI
- Add and remove routes dynamically
- Expose MCP tools and skills for agent integration
- Agent can manage your entire dev environment

<Note>
Your machine, your control. Everything runs locally with no external dependencies.
</Note>

## How it works

1. **Start the proxy**: OpenEnvX runs a local proxy server on port 1355
2. **Register services**: Use `openenvx proxy run` to start your app and register a route
3. **Access via subdomain**: Your app is now available at `https://myapp.localhost:1355`
4. **Multiple services**: Run multiple apps, each gets its own subdomain on the same port

The proxy automatically forwards requests from `myapp.localhost` to your app's actual port (like 3000, 8080, etc.).
